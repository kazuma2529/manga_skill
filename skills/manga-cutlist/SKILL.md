# manga-cutlist: 文章 → カットリスト変換

原作テキストを、漫画の最小単位「カット（コマの構成要素）」に分解する。
この工程が漫画制作パイプライン全体の土台になるため、最も丁寧に行うこと。

## 入力

ユーザーから提供される原作テキスト（数千字程度のエッセイ、物語、記事など）。

## 出力フォーマット

Markdownで以下の構造を出力する:

```markdown
# カットリスト

## 第1章：[章タイトル] - [サブタイトル]

1. [カットの内容（簡潔な1行）]
2. キャラ名「セリフがあればここに」
3. [描写・ナレーション]

---

## 第2章：[章タイトル] - [サブタイトル]

4. [カットの内容]
...

---

## カット総数: [N]カット

## 構成メモ

- 第1-2章: [セクション名]（[N]カット）
- 第3-5章: [セクション名]（[N]カット）
...

全体で[N]カット → ページ数は[N]ページ程度になる見込み
```

## 変換ルール

### 章の分割
- 物語の論理的な区切り（場面転換、時間経過、テーマ転換）ごとに章を設ける
- 章タイトルには内容がひと目で分かるサブタイトルを付ける
- 導入 → 展開 → クライマックス → 結末 の大きな流れを意識する

### カットの粒度
- 1カット = 1コマの構成要素（1つの動作、1つの表情、1つのセリフ）
- セリフがあるカットは `キャラ名「セリフ内容」` の形式で記述する
- 独白・心の声は `キャラ名（独白）「内容」` で区別する
- 背景・風景のみのカットも明示する（場面転換の導入に有効）
- 1つのカットに情報を詰め込みすぎない

### 場面転換の処理（最重要）
- 場面が変わる場合は、必ず以下のいずれかを挿入する:
  - ブリッジコマ（風景カット、時間経過を示す小物など）
  - ナレーション/モノローグカット（「それから数日後...」等）
  - 対比カット（過去と現在を並べるなど）
- 場面転換なしに突然別シーンに飛ぶと、読者が混乱する
- 回想の開始/終了は明確にわかるカットを入れる

### ボリューム管理
- 目安: 1ページあたり3〜5コマ → 15〜20ページなら全体60〜100カット
- 見せ場（感情が最高潮のシーン）は複数カットを使って丁寧に描く
- テンポよく進めたい箇所は1カットで複数の情報をまとめてもよい
- カット総数とページ数の見込みを末尾に記載する

### 構成メモ
- 章をグループ化し、物語全体のバランスを俯瞰するメモを付ける
- 導入が長すぎないか、クライマックスに十分なカットがあるか確認する

## 注意事項

- 原作テキストの内容を勝手に改変しない。省略はOKだが捏造はNG
- セリフは原作の言い回しを活かしつつ、吹き出しに収まる長さに調整する
- 登場人物名は原作に合わせて統一する
- 「画像」「写真」等のメディア参照は無視し、テキスト内容のみからカットを構成する

## ファイル出力仕様（重要）

このスキルは、**チャットにカットリスト本文を表示せず、`output/` 配下に必ずファイルとして保存する** ことを最優先とする。

### 作品タイトルからファイル名プレフィックスを決める

- ユーザーは入力テキストの先頭付近に、次のような形式で作品タイトルを含めていると想定する:
  - `タイトル: 余命10年の僕が君に伝えたいこと`
- この行から `タイトル:` 以降の文字列を取り出し、前後の空白を除去して **作品タイトル文字列** とする
- 作品タイトル文字列をもとに **`story_id`（ファイル名プレフィックス）** を決める:
  - Windowsファイル名として使えない文字（`/ \ : * ? " < > |`）はすべて `_` に置換する
  - 先頭と末尾の空白やピリオドは削除する
  - それ以外の文字（日本語・英数字・記号）は基本的にそのまま利用してよい
- `タイトル:` 行が見つからない場合のフォールバック:
  - `story_id = "untitled"` とし、後述のタイムスタンプで一意性を担保する

### ディレクトリ構成

- **最新版（Python等が参照する想定のファイル）**:
  - ベースディレクトリ: `output/cutlist/`
  - パス: `output/cutlist/{story_id}.md`
  - 同じ `story_id` で再実行した場合、ここは **上書きして最新状態を保つ**。
- **履歴（すべて残すアーカイブ）**:
  - ベースディレクトリ: `output/history/cutlist/`
  - 作品ごとのディレクトリ: `output/history/cutlist/{story_id}/`
- スキルは、これらのディレクトリが存在しない場合には **自動的に作成** してからファイルを書き込むこと。

### ファイル名とバージョン管理

- 履歴用には、1回の実行ごとに **タイムスタンプ付きのユニークなファイル名** を付けることで、**絶対に上書きしない**。
- タイムスタンプはローカル時間で `YYYYMMDD-HHMMSS` 形式を推奨する（例: `20260303-235959`）。
- カットリスト Markdown のパスは次の2系統とする:
  - 最新版: `output/cutlist/{story_id}.md`
  - 履歴: `output/history/cutlist/{story_id}/{story_id}_cutlist_{YYYYMMDD-HHMMSS}.md`
- 例:
  - 作品タイトル: `タイトル: 余命10年の僕が君に伝えたいこと`
  - story_id: `余命10年の僕が君に伝えたいこと`
  - ファイルパス:
    - 最新版:
      - `output/cutlist/余命10年の僕が君に伝えたいこと.md`
    - 履歴:
      - `output/history/cutlist/余命10年の僕が君に伝えたいこと/余命10年の僕が君に伝えたいこと_cutlist_20260303-235959.md`

### チャット出力ポリシー

- カットリストの **本文（Markdown）はチャットに出力しない**。
- 代わりに、次のような **保存結果のサマリーだけを簡潔にチャットに返す**:
  - 作品タイトル
  - 生成したカット総数・想定ページ数
  - 保存したファイルパス（1行）
- 例:
  - `カットリストを生成し、output/cutlist/余命10年の僕が君に伝えたいこと/余命10年の僕が君に伝えたいこと_cutlist_20260303-235959.md に保存しました。`

### 実装上の注意

- Cursorのファイル書き込み機能（ファイル追加・更新ツール）を用いて、上記パスにMarkdownファイルを書き込むこと。
- 生成したカットリストの内容と、保存されたファイルの内容が完全に一致していることを前提とする。
- 既存ファイルと同じパスを生成しないよう、**タイムスタンプまたは一意なID** を必ず含める。
