# manga-storyboard: カットリスト → 字コンテ変換

カットリストの各カットに「演出指示」を肉付けし、ページ・コマ単位の字コンテに変換する。
漫画家の「ネーム」に相当する工程。この先の工程に進むと修正が困難なため、ここで内容を確定させる。

## 入力

manga-cutlist スキルで生成されたカットリスト（Markdown形式）。

## 出力フォーマット

```markdown
# 字コンテ

---

## [N]ページ目：[ページタイトル]

### 1コマ目（[位置]・断ち切り:[方向]）

カット[N]+[N]

- **画角: [ショットタイプ]**
- 背景: [背景の説明]
- [キャラ名]: [位置], [表情/感情], [ポーズ]
- セリフ:「[セリフ内容]」
- ナレーション:「[ナレーション内容]」

### 2コマ目（[位置]・断ち切りなし）

カット[N]

- **画角: [ショットタイプ]**
...

---

# 構成サマリー

| ページ | 内容 | コマ数 | カット |
| ------ | ---- | ------ | ----- |
| 1      | ...  | 3      | 1-5   |
...

**合計: [N]ページ、[N]コマ、[N]カット**
```

## 演出の語彙

### ショットタイプ（画角）
- **顔アップ**: 顔全体〜首まで。感情表現の見せ場に使う
- **バストアップ**: 胸から上。最も汎用的。会話シーン向き
- **膝上**: 膝から上。ポーズや衣装を見せたい時
- **ロングショット**: 全身+背景。場所の説明、孤独感、登場シーン
- **ワイドショット**: 広い空間。風景、場面転換、群衆
- **極端なアップ**: 目や手元など一部分のみ。インパクト重視

### コマの位置
- **top**: ページ上部
- **middle**: ページ中央部
- **bottom**: ページ下部
- 左右分割: **middle-right**, **middle-left**, **bottom-right**, **bottom-left**

### 断ち切り（bleed）
- ページの端まで絵を拡張する指定
- 方向: top, bottom, left, right の組み合わせ
- 見せ場のコマや情景コマで使用する
- 通常のコマは「断ち切りなし」

### 見せ場コマ
- `【見せ場】` タグを付ける
- 画角に `（大きめ）` を付加する
- 物語のクライマックス、感情的ピーク、重要なセリフに使用
- 1ページに1つまでが目安

## 変換ルール

### ページ構成
- 1ページあたり 3〜5コマ が基本
- 見せ場は大きなコマ（2〜3カット分を1コマに統合）
- テンポを上げたい箇所は小さいコマを連続させる（4〜5コマ）
- 余韻を残す場面は少ないコマ数（2〜3コマ）にする

### カットの統合
- 関連する複数カットを1コマにまとめてよい（例: カット15+16）
- 統合する場合は `カット[N]+[N]` と明記する
- 1コマに無理に詰め込まない。情報量が多い場合は分割する

### セリフとナレーション
- **セリフ**: キャラが声に出して言う言葉。吹き出しに入る
- **ナレーション**: 語り手の声。四角い枠に入る。回想や状況説明に使う
- **心の声**: 思考や独白。雲型の吹き出し
- 1コマにセリフは最大2つ、ナレーションは最大2つまで

### 感情と表情の指定
- 具体的に書く: ×「悲しい」→ ○「涙をこらえて唇を噛む」
- キャラの向き（facing）も指定: 正面、斜め、横顔、背面

### ページの流れ
- 右から左、上から下に読む（日本式マンガ）
- 1コマ目は右上、最終コマは左下が基本配置
- 見開きの流れを意識する（奇数ページが右、偶数ページが左）

## 構成サマリー

末尾に全ページの一覧表を付ける。ページ番号、内容概要、コマ数、対応カット番号を記載。
全体のページ数・コマ数・カット数の合計も記載する。

## 注意事項

- カットリストの内容を飛ばさない。全カットをいずれかのコマに割り当てる
- 同じキャラの連続するバストアップが続かないよう、画角にバリエーションを持たせる
- ユーザーが「ここを直したい」と言ったら柔軟に修正する。これがブラッシュアップの最後の機会

## ファイル出力仕様（重要）

このスキルは、**チャットに字コンテ本文を表示せず、`output/` 配下に必ずファイルとして保存する** ことを最優先とする。

### 作品タイトルと story_id の取得

- 入力として与えられるカットリスト（あるいはユーザーの追記テキスト）には、次のような行が含まれていると想定する:
  - `タイトル: 余命10年の僕が君に伝えたいこと`
- この行の `タイトル:` 以降を取り出し、前後の空白を除去して作品タイトル文字列とする。
- 取得した作品タイトルから、`manga-cutlist` と同じルールで `story_id` を決定する:
  - Windowsファイル名として使えない文字（`/ \ : * ? " < > |`）は `_` に置換
  - 先頭・末尾の空白やピリオドを削除
  - それ以外の文字は基本そのまま利用
- `タイトル:` 行が見つからない場合:
  - `story_id = "untitled"` とし、後述のタイムスタンプで一意性を担保する

### ディレクトリ構成

- **最新版（ユーザーが直接開く想定のファイル）**:
  - ベースディレクトリ: `output/storyboard/`
  - パス: `output/storyboard/{story_id}.md`
  - 同じ `story_id` で再実行した場合、ここは **上書きして最新状態を保つ**。
- **履歴（すべて残すアーカイブ）**:
  - ベースディレクトリ: `output/history/storyboard/`
  - 作品ごとのディレクトリ: `output/history/storyboard/{story_id}/`
- ディレクトリが存在しない場合には、自動的に作成してからファイルを書き込むこと。

### ファイル名とバージョン管理

- 履歴用には、1回の実行ごとに **タイムスタンプ付きのユニークなファイル名** を付け、**絶対に上書きしない**。
- タイムスタンプ形式は `YYYYMMDD-HHMMSS` を推奨する。
- 字コンテ Markdown のパスは次の2系統とする:
  - 最新版: `output/storyboard/{story_id}.md`
  - 履歴: `output/history/storyboard/{story_id}/{story_id}_storyboard_{YYYYMMDD-HHMMSS}.md`
- 例:
  - 作品タイトル: `タイトル: 余命10年の僕が君に伝えたいこと`
  - story_id: `余命10年の僕が君に伝えたいこと`
  - ファイルパス:
    - 最新版:
      - `output/storyboard/余命10年の僕が君に伝えたいこと.md`
    - 履歴:
      - `output/history/storyboard/余命10年の僕が君に伝えたいこと/余命10年の僕が君に伝えたいこと_storyboard_20260303-235959.md`

### チャット出力ポリシー

- 字コンテ本文（各ページ・各コマの詳細なMarkdown）は **チャットに出力しない**。
- 代わりに、次のような **構成サマリーと保存結果のみ** を簡潔に返す:
  - 作品タイトル
  - ページ数・総コマ数・総カット数
  - 保存した字コンテファイルのパス
- 例:
  - `字コンテを生成し、全15ページ・合計52コマ・80カットとして構成しました。output/storyboard/余命10年の僕が君に伝えたいこと/余命10年の僕が君に伝えたいこと_storyboard_20260303-235959.md に保存しました。`

### 実装上の注意

- `manga-cutlist` と同じ story_id を使うことで、後続の `manga-art-prompt` や `manga-panel-layout` から作品単位でファイルを関連付けやすくする。
- Cursorのファイル書き込み機能を用いて、上記パスにMarkdownファイルを書き込むこと。
- 既存ファイルとの衝突を避けるため、タイムスタンプなどの一意なIDを必ずファイル名に含める。

## このバージョンでの保存ポリシー

- `output/history` 配下には新しいファイルやディレクトリを一切作成しないこと。
- 字コンテの保存先は常に `output/storyboard/{story_id}.md` とする。
- 同じ `story_id` で再実行した場合は、このファイルをそのまま上書き保存する。
